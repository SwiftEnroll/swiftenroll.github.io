## Hugo Template Rules

**Verify against Hugo documentation (https://gohugo.io/documentation/) before making template changes.** This is non-negotiable. Do not rely on assumptions about Go template behavior.

### Known Gotchas

1. **`$` is always the top-level page context**, not the current range/with item. Inside `{{ range .Pages }}` then `{{ with .Params.image }}`, `$` refers to the list page, not the iterated post. Capture variables before entering `with` blocks:
   ```
   {{ range .RegularPages }}
     {{ $page := . }}
     {{ with $page.Params.image }}
       <a href="{{ $page.RelPermalink }}">...</a>
     {{ end }}
   {{ end }}
   ```

2. **`.TableOfContents` renders its own `<nav id="TableOfContents">`**. Do not wrap it in another `<nav>` — use a `<div>` wrapper if needed.

3. **`.Pages` vs `.RegularPages`**: `.Pages` includes subsection `_index.md` files. Use `.RegularPages` when listing content (e.g., blog posts).

4. **`default` function**: `{{ .Params.foo | default "bar" }}` works correctly — the pipe passes the value (even if nil) to `default`. No need to assign to a variable first.

5. **Hugo's RSS**: Hugo auto-generates RSS feeds for sections. Use `{{ with .OutputFormats.Get "rss" }}` to link them. Don't manually build feed XML.

### Template Conventions

- **Reuse partials** — check `layouts/partials/components/` before writing inline markup. The shared CTA (`components/cta.html`), image optimizer (`components/optimized-image.html`), and other partials exist for a reason.
- **Reuse shortcodes** — check `layouts/shortcodes/` before creating new content patterns. There are 24 shortcodes already.
- **Config belongs in `hugo.toml`** — never hardcode URLs, colors, button text, or CTA copy in templates. Pull from `{{ .Site.Params }}`.
- **Front matter fields**: blog posts use `image` (not `featured_image`) for the hero image.

## Tailwind CSS

- **Utility-first** — use Tailwind classes, not custom CSS (except for established utilities in `assets/css/main.css`)
- **Color tokens**: `primary-*`, `secondary-*`, `buttonPrimary` — see `tailwind.config.js`
- **Build output**: `assets/css/main.css` → `static/css/style.css`
- **`hugo_stats.json`** is generated by `writeStats = true` but is not consumed by the Tailwind build (Tailwind scans layout/content files directly via `tailwind.config.js`). It creates noisy diffs.

## PR Checklist

1. `npm run build` passes
2. `just lychee` — no broken links
3. `just lighthouse build` — meets thresholds
4. Responsive check: 375px mobile + 1280px desktop
5. No console errors, no broken images, no horizontal scrollbars
6. Documentation updated if needed (see `/docs/engineering/DOCUMENTATION.md`)
7. Screenshots uploaded to GitHub release assets (never commit images to repo)
