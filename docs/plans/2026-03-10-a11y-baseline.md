# Accessibility Baseline Implementation Plan

> Autonomous execution — no human review gate.

**Goal:** Improve site accessibility to Lighthouse a11y score >= 98 by adding ARIA to FAQ toggles, a skip-to-content link, `<main>` landmark, visible focus states, and nav aria-labels.

**Architecture:** Pure HTML/CSS/JS changes across Hugo layout templates and CSS. No new dependencies.

**Tech Stack:** Hugo templates, Tailwind CSS utility classes, vanilla JS

**Planning Decisions:**
- FAQ IDs use `{{ $i }}` index — unique within a page if one FAQ per page (typical for marketing site). If multiple FAQs appear on one page, IDs would collide; noted in PR.
- Skip link target is `#main-content` — main content div will be converted to `<main id="main-content">`.
- Focus states: btn-primary/btn-outline already have `focus-visible:ring-2`; adding global rules for `a` and `button` elements.
- Footer `<ul>` lists are NOT wrapped in `<nav>` — footer is already a `<footer>` landmark and column titles (`h3`) provide context; adding nav wrappers would over-engineer.
- Mobile nav toggle (CSS checkbox) gets `aria-label` on the `<label>`.

## Tasks

### Task 1: Add skip-to-content link and `<main>` landmark in baseof.html
- File: `layouts/_default/baseof.html`
- Add `<a href="#main-content" class="skip-to-content">Skip to content</a>` as first element inside `<body>`
- Change `<div class="pt-20 w-full min-w-0">` → `<main id="main-content" class="pt-20 w-full min-w-0">`
- Change closing `</div>` → `</main>`
- Verify: `npm run build` passes

### Task 2: Add ARIA attributes to FAQ shortcode
- File: `layouts/shortcodes/faq.html`
- Use `{{ range $i, $q := $data.questions }}` for indexed iteration
- Add `id="faq-btn-{{ $i }}"`, `aria-expanded="false"`, `aria-controls="faq-answer-{{ $i }}"` to each `<button>`
- Add `id="faq-answer-{{ $i }}"` and `role="region"` and `aria-labelledby="faq-btn-{{ $i }}"` to each content div
- Replace inline onclick with call to `toggleFaq(this)` function
- Add `<script>` block with `toggleFaq()` that also sets `aria-expanded`
- Verify: `npm run build` passes

### Task 3: Add nav aria-label to header
- File: `layouts/partials/layout/header.html`
- Add `aria-label="Main navigation"` to the `<nav>` element
- Add `aria-label="Open navigation menu"` to the mobile `<label for="nav-toggle">`
- Verify: `npm run build` passes

### Task 4: Add skip-to-content CSS and global focus-visible styles
- File: `assets/css/main.css`
- Add `.skip-to-content` styles in `@layer components`
- Add global `a:focus-visible`, `button:focus-visible` in `@layer base`
- Verify: `npm run build` passes

### Task 5: Commit all changes
- `git add -A && git commit -m "feat(a11y): ARIA on FAQ, skip link, main landmark, nav labels, focus styles"`
