# Architecture Overview

## System Overview

SwiftEnroll's marketing website is a static site generated by Hugo. Content is authored in Markdown, styled with Tailwind CSS, and deployed to GitHub Pages.

## Data Flow

### 1. Content Authoring
**Input:** Markdown files in `content/` with Front Matter
- Front Matter defines metadata (title, description, layout, etc.)
- Content body written in Markdown
- Supports Hugo shortcodes for reusable components

### 2. Template Processing
**Hugo Layouts** in `layouts/` process Markdown into HTML
- `_default` layouts for standard page types
- `partials` for reusable components
- Go template syntax for dynamic content

### 3. Style Compilation
**Tailwind CSS** scans HTML/content for utility classes
- Source: `assets/css/main.css`
- Tailwind processes and generates optimized CSS
- Output: `static/css/style.css`

### 4. Build Process
**`hugo --minify`** generates static HTML
- Combines content + layouts + styles
- Minifies HTML, CSS, JS
- Outputs complete site to `public/`

### 5. Deployment
**GitHub Pages** serves static files
- Contents of `public/` directory
- Deployed via GitHub Actions on push to main
- Available at https://swiftenroll.github.io/

## Integrations

### Forms
- Direct POST to `submit-form.com` endpoints
- No server-side code in this repository
- Configuration: `hugo.toml` -> `[params.forms]`

### Analytics
- Google Analytics 4 (GA4)
- Tracking ID in `hugo.toml`
- Client-side JavaScript

### Anti-Spam
- Cloudflare Turnstile on forms
- Honeypot fields (hidden form inputs)

### Search
- None (static site)
- No search backend required

## Configuration Management

### Single Source of Truth
**`hugo.toml`** contains all site configuration:
- Site metadata (title, baseURL, description)
- Menu structure
- Global parameters (company info, social links, form endpoints)
- Build settings
- Analytics IDs

### Environment Variables
Hugo supports environment variable overrides:
- `HUGO_PARAMS_...` for local development overrides
- Example: Disable form submissions locally

## Performance Characteristics

### Build Time
- Fast static site generation (seconds)
- Incremental builds in development mode

### Page Load
- No server-side processing
- Static files served directly from CDN
- Minimal JavaScript (forms, analytics only)

### Scalability
- Unlimited scalability (static files)
- No backend bottlenecks
- CDN-friendly
